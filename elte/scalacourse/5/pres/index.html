implicit keyword

Implicit resolution

Szabalyok

  • Marking: csak az implicitnek jelolt entitasok

  • Scope: az implicitnek egyszeru azonositokent a scope-ban kell lennie

  • Egyszerre egy: a compiler csak egy implicitet alkalmaz

  • Explicitek elore: ha van explicit ertek megadva, nincs implicit alkalmazas

  • Elofordulas: konverzio egy elvart tipusra, konverzio egy apply fogadasara vagy implicit parameterek

  • Elnevezes: az implicit neve nem szamit

Impicit feloldas a gyakorlatban

Amikor a compiler egy tipushibat vagy egy applikacios hibat talal

Megnezi a scope-ban levo impliciteket

Megprobalja "kijavitani" a hibat

Magas komplexitas

Lassu, az elerheto implicitek szamaval aranyosan lassul

Implicit parameterlistak

Egy parameterlistat lehet implicitnek jelolni

def apply(x: Int)(implicit y: Int)
				
impicit val myInt = 3
apply(2)
apply(2)(4) // explicit nyer

Lehet akar konstruktor is (osztaly es masodlagos)

class Addition(val x: Int)(implicit private val y: Int) {
    private val result = x + y
    def printer() = println(result)
}

implicit val operand = 4
val a = new Addition(4)
a.printer()

A parameterlistaban lehetnek varok, valok es defek

Implicit fuggvenyek

implicit def int2String(i: Int): String
class A(val x: Int)
class B(val y: Int)

implicit def a2B(v: A): B = new B(v.x)

def printer(s: B) = println(s.y)

printer(new A(3)) // => printer(a2B(new A(3)))

Ajanlott nevezektan: tipusA2TipusB

A signatura minding A => B, tobb parameterrel nincs ertelme

De tuple-okkel mukodik:

class Aa(val x: Int)
class Ab(val x: Int)
class B(val y: Int)

implicit def a2B(a: (Aa, Ab)): B = new B(a._1.x + a._2.x) // Csunya, mindig hasznaljuk az unapplyt!

def printer(s: B) = println(s.y)

printer(new Aa(3), new Ab(2)) // Nem is kell explicit tuple

Implicit osztalyok

Scala 2.10 ota

implicit kulcsszo a class elott

Az elsodleges konstruktor reszt vehet implicit konverzioban

Megszoritasok

  • Egy kulso trait/class/object-ben kell oket definialni

  • Csak egy nemimplicit parameteruk lehet

  • Egyeduli szimbolumnak kell lenniuk ezzel a nevvel a scope-ban

Implicit osztalyok

object Helpers {
  implicit class IntWithTimes(x: Int) {
    def times[A](f: => A): Unit = {
      def loop(current: Int): Unit =
        if(current > 0) {
          f
          loop(current - 1)
        }
        loop(x)
      }
    }
}

import Helpers._ // Hozzuk a scope-ba

5 times println("HI")

Szintaktikai cukorka

Ebbol:

implicit class RichInt(n: Int) extends Ordered[Int] {
  def min(m: Int): Int = if (n <= m) n else m

Ezt generalja a fordito:

class RichInt(n: Int) extends Ordered[Int] {
  def min(m: Int): Int = if (n <= m) n else m
}

implicit final def RichInt(n: Int): RichInt = new RichInt(n)

Gyakori volt egy uj tipus letrehozasa az implicit konverter melle

View bounds

Eloirja, hogy egy A tipust hasznalhassuk B-kent

Legyen hozza implicit konverter a hivas helyen

Nem osszekeverendo a context bounds-dzal

def printer2[A <% B](a: A) = println(a.y)

Viszonylag ritkan hasznalt feature

Scala 2.10 ota deprecated

Majdnem az osszes felhasznalasa kivalthato context bounds-dzal

Context bounds

Eloirja, hogy egy A tipushoz legyen egy B[A] tipusu ertekunk

Tipikus szintaxis:

def m[A: B](a: A) {
                        
    ...
    implicitly[B[A]].methodOnB // Itt hivjuk az implicit konvertert
}

Kifejtve:

def m[A](a: A)(implicit ev: B[A]) {
    ...
    a.methodOnB
}

ev: evidence roviditese, ez ugyanis a bizonyitek, hogy A-t B[A]-kent tudjuk hasznalni

A masodik valtozat az ajanlott
